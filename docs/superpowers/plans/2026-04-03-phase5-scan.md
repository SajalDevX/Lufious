# Phase 5: Scan (Camera + AI Plant ID)

**Goal:** Replace the Scan placeholder with a real camera screen, a mock AI identification flow, and a result screen with "Add to Garden" pre-fill.

**Architecture:**
- CameraX (camera-camera2, camera-lifecycle, camera-view) + concurrent-futures-ktx for async setup
- `ScanDataSource.scanPlant(imageBytes)` — mock AI with 1.5 s delay; swappable for real API later
- Scan results saved to Firestore `users/{uid}/scans/{scanId}` to support scan history
- `ScanResultScreen` loads by scanId via `SavedStateHandle`
- `AddPlant` route gains optional `?species={species}` query param so ScanResult can pre-fill it
- All sub-screens (ScanResult) live in inner NavHost (bottom nav visible)
